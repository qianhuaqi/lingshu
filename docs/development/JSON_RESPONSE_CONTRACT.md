# JSON Response Contract

Status: P3-01 baseline
Issue: #92

## API

P3-01 adds one narrow response factory:

```python
Response.json(value, *, status=200, headers=())
```

The factory uses Python's standard library JSON encoder and returns a regular buffered `Response`.

## Serialization contract

`Response.json` serializes with:

```text
ensure_ascii=False
allow_nan=False
separators=(",", ":")
encoding=utf-8
```

This keeps UTF-8 output readable, rejects non-finite floats such as `NaN`, and avoids whitespace differences in generated bytes.

Serialization failures are raised before a `Response` is created. P3-01 does not introduce a schema or validation layer.

## Header contract

If the caller does not provide `content-type`, the response sets:

```text
application/json; charset=utf-8
```

If the caller provides `content-type`, LingShu preserves the caller value and does not add a second content type.

`content-length` is still added during `prepare()` / `commit()` using the existing buffered response behavior.

## Non-goals

P3-01 does not add:

- request-body JSON helpers;
- content negotiation;
- validation or schema support;
- dependency injection;
- OpenAPI;
- database, cache, storage, auth, tenant, or RBAC integration;
- new runtime dependencies;
- public package publication;
- production-readiness claims.
